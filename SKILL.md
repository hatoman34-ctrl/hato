# Studio Namma 风格中文网站生成技能

## 何时使用

当用户希望生成以下类型网站时使用本技能：

- 个人设计师作品集
- 品牌设计工作室官网
- 创意工作室官网
- Webflow / no-code 开发工作室官网
- 高级视觉作品展示页
- 大胆、极简、高冲击力的品牌网站

## 核心目标

生成一个借鉴 Studio Namma 设计语言、但内容原创的中文网站。

重点不是复制原站，而是复用这些设计原则：

- 大字号标题
- 黑白高对比
- 作品图像主导
- 简短锋利文案
- 服务能力矩阵
- 清晰的项目流程
- 强烈联系 CTA
- 适度滚动与 hover 动效

## 必读文件

执行前优先读取：

- `design.md`：完整设计分析与视觉规范
- `AGENT.md`：代理执行规则、页面结构、验收标准

如果当前项目有 `images/` 文件夹，必须优先使用其中图片作为作品素材。

## 默认页面结构

生成单页网站时，默认包含：

1. 顶部导航
2. Hero 大标题
3. 品牌宣言
4. Selected Projects
5. Services
6. Approach
7. Contact
8. Footer

不要先做营销落地页式说明，而要直接呈现作品集或工作室官网体验。

## 视觉原则

### 字体

- 标题必须足够大
- 行高紧凑
- 字距为 0
- 中文使用黑体类字体
- 英文可以大写或局部穿插

### 色彩

使用克制基础色：

- 黑
- 白
- 浅灰
- 深灰

强调色来自作品图，不做无意义渐变背景。

### 作品图

作品图必须是视觉主体。

禁止：

- 只放小缩略图
- 只用灰色占位
- 让文字解释压过作品

推荐：

- 大图横排
- 图片墙
- 视频预览
- hover 露出项目名
- 项目名、年份、类型只保留高密度信息

## 文案风格

文案要短、硬、有态度。

可使用这些中文语气：

- 不是做一个网站，是让品牌被记住。
- 平庸不是选项。
- 每个像素都要有理由。
- 好看只是开始，转化才是结果。
- 静态是骨架，动态是记忆。
- 你的品牌，我们的游乐场。

避免：

- 一站式解决方案
- 赋能增长
- 专业高效
- 数字化转型
- 以客户为中心

## 流程模板

默认流程：

1. Kick-off：对齐目标与范围
2. Research：研究品牌、用户与市场
3. Design：建立视觉方向与页面系统
4. Development：把设计变成可运行网站
5. Delivery：交付、培训与上线
6. Optimisation：上线后持续优化

## 实现建议

如果是静态网站：

- 使用 `index.html`
- 使用 `styles.css`
- 使用 `script.js`
- 不引入大型框架

如果已有框架：

- 沿用现有技术栈
- 不重构无关文件
- 只改与页面相关的组件和样式

## 质量检查

交付前检查：

- 首屏是否足够强
- 标题是否像设计工作室，而不是普通企业官网
- 图片是否真实显示
- 移动端是否没有文字重叠
- 页面是否没有横向溢出
- CTA 是否明显
- 作品、服务、流程是否完整
- 文案是否避免空泛套话

## 输出格式

完成后向用户说明：

- 生成了哪些文件
- 如何打开或运行
- 哪些内容可继续替换为真实作品
- 是否完成桌面端与移动端检查
