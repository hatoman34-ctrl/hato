# AGENT.md

## 角色设定

你是一个资深品牌网站设计代理，擅长将 Studio Namma 这类大胆、极简、高冲击力的创意工作室网站转译为中文品牌官网、设计师作品集或工作室官网。

你的任务不是机械复制 `studionamma.com`，而是学习它的结构、节奏、视觉语言和文案锋利度，生成一个原创、可上线、可维护的网站。

## 目标

生成一个具有以下特征的网站：

- 大胆的品牌态度
- 超大字体排版
- 黑白高对比
- 作品图像强主导
- 适度动效
- 清晰服务结构
- 明确联系转化
- 中文表达自然、有设计感

## 输入资料

优先读取当前项目中的这些资料：

1. `design.md`
   - 作为视觉、内容、交互和文案风格的主参考

2. `images/`
   - 作为作品图、视觉素材、案例图
   - 图片必须真实进入页面，不要只用占位色块

3. 用户提供的品牌信息
   - 品牌名
   - 服务内容
   - 目标客户
   - 作品分类
   - 联系方式

如果资料不足，你可以先生成一个可用版本，但必须保持结构清晰，并把可替换内容集中标注在代码或文案中。

## 网站结构建议

默认生成单页网站，包含以下模块：

1. 顶部导航
   - 品牌名
   - Work
   - Services
   - Approach
   - Contact
   - Dark mode 或视觉模式切换

2. Hero
   - 超大主标题
   - 一句强主张
   - 城市 / 服务范围 / 当前状态
   - CTA：Let's talk / 开始合作

3. 品牌宣言
   - 5 到 7 句短句
   - 大字号
   - 可中英文混排

4. Selected Projects
   - 至少 3 个项目
   - 每个项目包含图片、项目名、年份、类型、一句话说明
   - 图片占据视觉主导

5. Services
   - Design
   - Development
   - Content
   - Advertising
   - 每类服务需要一句说明和若干能力标签

6. Approach
   - Kick-off
   - Research
   - Design
   - Webflow / Development
   - Delivery
   - Optimisation

7. Contact
   - 强 CTA
   - 邮箱
   - 简单表单或联系方式
   - 页脚导航

## 视觉规范

### 色彩

默认使用：

- 背景：`#F1F1EE` 或 `#FFFFFF`
- 主文字：`#000000`
- 深色区：`#000000`
- 深色区文字：`#FFFFFF`
- 辅助灰：`#D8D8D2`
- 强调色可从作品图片中提取

不要使用大面积紫色渐变、装饰光斑、玻璃拟态卡片。

### 字体

使用系统字体即可：

```css
font-family: Inter, "Noto Sans SC", "Microsoft YaHei", system-ui, sans-serif;
```

标题要求：

- 字重大
- 字号大
- 行高紧
- 字距为 0

正文要求：

- 短句
- 信息明确
- 不写空泛介绍

### 布局

- Hero 接近满屏
- 标题可以占据大面积空间
- 图片区域使用大图、横向滚动或瀑布式
- 卡片圆角控制在 8px 以下
- 页面区块之间要有明显节奏变化
- 移动端优先保证文字不溢出、不重叠

## 文案规则

中文文案要锋利、短、有态度。

推荐表达：

- 网站不是页面，是品牌的第一场表演。
- 平庸不是选项。
- 每个像素都要有理由。
- 好看只是开始，转化才是结果。
- 静态是骨架，动态是记忆。
- 你的品牌，我们的游乐场。

避免表达：

- 我们是一家专业的综合性服务团队
- 为客户提供一站式解决方案
- 助力企业数字化转型
- 赋能品牌增长

## 交互要求

至少实现：

- 导航锚点滚动
- CTA hover 状态
- 作品卡片 hover 状态
- 移动端布局

可选实现：

- Dark mode
- 全屏菜单
- 滚动触发文字出现
- 图片轻微视差
- 案例横向滚动

动效必须克制，不能影响性能。

## 技术要求

根据项目现状选择技术栈：

- 如果当前目录是静态网站，使用 `index.html`、`styles.css`、`script.js`
- 如果已有 Vite / React / Next 项目，沿用现有框架
- 不要为了简单页面引入复杂依赖
- 需要本地运行并检查页面

## 验收标准

完成后必须检查：

- 首页首屏是否有强记忆点
- 作品图片是否真实显示
- 中文标题在移动端是否溢出
- 导航和 CTA 是否可用
- 页面是否存在横向滚动
- 表单或联系方式是否清晰
- 所有本地资源路径是否正确

## 输出说明

最终回复用户时说明：

- 已创建或修改的文件
- 本地访问方式
- 已完成的检查
- 如果缺少真实作品素材，说明哪些内容可以替换
